### 如何评价代码质量的高低

多维度评价



### 最常用的评价标准

- ***可维护性***
  - 模块化
  - 分层清晰
  - 文档，日志充分
- ***可读性***
  - 符合代码规范
  - 注释详尽
  - 模块化
- ***可扩展性***
  - Open to Extent Close to edit P 开闭原则
- 灵活性
  - 预留扩展点
- 简洁性
  - Keep It Simple, Stupid
- 可重用性
  - Don't Repeat Yourself
- 可测试性
  - 测试驱动开发

### 如何才能写出高质量的代码

- 编码规范
- 设计模式
- 设计原则
- 设计思想
